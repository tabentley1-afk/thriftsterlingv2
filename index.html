Thrive on Thrift — Donation Pickup & Scheduling System

A donation pickup management system built for Thrive on Thrift, a nonprofit thrift store in Sterlington, LA. It gives donors an online way to request a pickup, and gives staff a dashboard to schedule the van, price items, and track every ticket from request to completion.

Live Site ↗ Source Code ↗ Run It Locally ↓
Overview

What this system does

Before this project, pickup requests for Thrive on Thrift were handled by phone and memory. The DMS replaces that with a public donor form, an internal ticket queue, automatic mileage and labor-cost estimates, a drag-and-drop van calendar, blackout-day handling, and CSV export for recordkeeping — all built around Central Time business hours.

It's a server-rendered Node.js app (Express + EJS), backed by SQLite, deployed on Render. The Google Maps Distance Matrix API recalculates round-trip mileage and drive time automatically whenever staff open a ticket.

Feature Tour

Walkthrough

Public

Donation Request Page

Donors schedule a free pickup online: contact info, pickup address, item categories, condition, notes, a preferred date and time, and up to 10 photos — so staff have what they need before approving a pickup.

GET /donate

Staff

Admin Tickets Dashboard

A central queue of every pickup request, with status, donor info, and preferred time at a glance, plus quick access to availability, the calendar, blackout days, reports, and CSV export.

GET /admin/tickets (session-protected)

Staff

Ticket Detail Page

One request in full: contact info, items, condition, notes, and photos, alongside auto-recalculated mileage and drive time, an editable crew size and fuel rate, live cost estimate, status control, and scheduling.

GET /admin/tickets/:id

Staff

Van Schedule

A running list of upcoming scheduled pickups — start time, end time, donor, and pickup address — so staff can plan the single pickup van without double-booking. Also available as a drag-and-drop month calendar.

GET /admin/availability · GET /admin/calendar

Staff

CSV Export

One-click export of every ticket — donor info, address, categories, condition, preferred time, mileage, drive time, crew size, and estimated cost — for offline reporting and recordkeeping.

GET /admin/export.csv

New

Reports

A summary dashboard: tickets by status and category, bags and furniture pieces donated, total round-trip miles, and total and average estimated cost — including a completed-pickups breakdown.

GET /admin/reports

Changelog

What was reviewed and fixed in this pass

A code review against this documentation turned up a few real issues. Here's what changed:

BUG

Reports link went nowhere

The admin dashboard already linked to /admin/reports, and the documentation already described a "reports" feature, but the route didn't exist. Added the route and a summary dashboard (tickets by status/category, totals, completed-pickup costs).

BUG

Van Schedule address was always blank

/admin/availability read a pickup_address column from the schedule table, but that table never stored an address — so the column always rendered empty. listScheduled() now joins against tickets so the real pickup address shows up.

SECURITY

.env and data.sqlite are tracked in git

The repository has a committed .env file containing a live Google Maps API key, plus a tracked data.sqlite database file, even though both are listed in .gitignore — they were committed before the ignore rule was added. Recommended: rotate the API key, then run git rm --cached .env data.sqlite and commit, so future donor data never lands in version control. See the README for the exact commands.

ADDED

Public homepage

/ previously just redirected straight to the donor form. It now renders an actual homepage — store story, how a pickup works, what's accepted, hours, and directions — so the live URL works as a real front door, not just a form.

Setup

Run it locally

git clone https://github.com/tabentley1-afk/thrift-sterlington.git
cd thrift-sterlington
npm install
cp .env.example .env        # then fill in GOOGLE_MAPS_API_KEY
npm run init-db
npm start                   # http://localhost:3000
VariablePurpose
GOOGLE_MAPS_API_KEYRequired. Powers automatic mileage/drive-time recalculation on ticket open.
ADMIN_SECRETPassword for /admin staff login. Defaults to password if unset — change this in production.
SESSION_SECRETSigns the admin session cookie.
FUEL_COST_PER_MILEUsed in cost estimates. Defaults to 0.20.
EMPLOYEE_HOURLYUsed in labor cost estimates. Defaults to 10.
DATA_DIR / UPLOAD_DIROptional. Persistent storage paths for the database and uploaded photos in production.
